# Skills Review

Claude Code スキルをベストプラクティスに照らして包括的にレビューし、具体的な改善提案を提供するスキルです。

## 概要

このスキルは、Claude Code のスキルファイル（SKILL.md）を 6 つの観点から評価し、A-F 評価とスコアを算出します。発見可能性、Progressive Disclosure、コンテンツ品質などのベストプラクティスに準拠しているかをチェックし、具体的な改善提案を提供します。

## インストール

### 前提条件

Claude Code CLI がインストールされている必要があります。

### マーケットプレースの追加

```bash
/plugin marketplace add interactive-inc/claude-plugins
```

### プラグインのインストール

```bash
/plugin install claude@interactive-claude-plugins
```

## 主な機能

- **包括的な評価**: 6 つの観点（Description 品質、Progressive Disclosure、コンテンツ品質、ワークフロー、テンプレート・例、技術的詳細）からスキルを評価
- **A-F 評価**: 0-100 点のスコアリングと A-F グレーディング
- **優先度付き改善提案**: Critical / High / Medium / Low の 4 段階で問題を分類
- **具体的な修正例**: Before/After 形式で改善例を提示
- **継続的改善サイクル**: 改善後の再レビューによる品質向上トラッキング

## 使用タイミング

このスキルは以下のような場合に使用します：

- スキルを新規作成した後の品質チェック
- 既存スキルの改善点を特定したい
- ベストプラクティスに従っているか確認したい
- スキルの発見可能性を向上させたい
- Progressive Disclosure が適切に実装されているか確認したい

## ファイル構成

```
skills/claude/skill-review/
├── README.md              # このファイル
├── SKILL.md              # メインスキル定義（実行フロー、評価基準）
├── CHECKLIST.md          # 詳細な評価チェックリスト（6カテゴリ、配点付き）
├── REPORT_TEMPLATE.md    # レビューレポート出力テンプレート
└── EXAMPLES.md           # 良い例・悪い例の具体例
```

## 使い方

### 基本的な使用方法

1. **スキルの呼び出し**
   ```
   「skill-review スキルを使って [スキル名] をレビューしてください」
   ```

2. **対象スキルの指定**
   - スキルディレクトリのパスを指定
   - 例: `skills/my-skill/`

3. **レビュー結果の確認**
   - 総合評価（A-F グレード、スコア）
   - 優先度別の改善提案
   - 具体的な修正例
   - 次のアクション

### 実行フロー

詳細な実行フローは [SKILL.md](./SKILL.md) を参照してください。

1. スキルファイルの読み込み
2. ベストプラクティスチェック
3. 問題の特定と分類
4. 改善提案の作成
5. 改善例の提示

## 評価基準

### 評価スコア

- **A（優秀）**: 90-100点 - すべてのベストプラクティスに準拠
- **B（良好）**: 70-89点 - ほとんどのベストプラクティスに準拠、軽微な改善点のみ
- **C（要改善）**: 50-69点 - いくつかの重要な問題あり
- **D（多くの問題）**: 30-49点 - 多数のベストプラクティス違反
- **F（不合格）**: 0-29点 - 致命的な問題あり、全面的な再作成を推奨

### 評価観点

詳細な評価項目と配点は [CHECKLIST.md](./CHECKLIST.md) を参照してください。

1. **Description 品質**: 三人称、具体的機能、トリガーワード、最適な長さ（200-800文字）
2. **Progressive Disclosure**: SKILL.md ≤500行、外部参照の活用、単層の深さ
3. **コンテンツ品質**: 簡潔性、時間非依存性、用語の一貫性
4. **ワークフロー**: 明確なステップ、検証ポイント、エラーハンドリング
5. **テンプレート・例**: 出力テンプレート、良い例・悪い例の提示
6. **技術的詳細**: スクリプト、MCP ツール参照、依存関係

### 優先度分類

- 🔴 **Critical**: 致命的な問題（スキルが機能しない）
- 🟠 **High**: 重要な改善（発見可能性や品質に大きく影響）
- 🟡 **Medium**: 推奨される改善（ベストプラクティスからの逸脱）
- 🟢 **Low**: 最適化（さらなる品質向上）

## レビューレポート

レビュー結果は以下の形式で出力されます：

1. **エグゼクティブサマリー**: 総合評価、スコア、主要な発見事項
2. **優先度別の改善提案**: 各問題の詳細と修正方法
3. **具体的な修正例**: Before/After 形式のコード例
4. **ベストプラクティスへの準拠状況**: 各カテゴリの評価
5. **推奨される次のアクション**: 優先順位付きのアクションアイテム

詳細なレポート形式は [REPORT_TEMPLATE.md](./REPORT_TEMPLATE.md) を参照してください。

## 良い例・悪い例

スキル記述の具体的な良い例と悪い例は [EXAMPLES.md](./EXAMPLES.md) を参照してください。

- Description の書き方
- Progressive Disclosure の実装
- ワークフローの定義
- テンプレートの提供
- など

## 注意事項

1. **対象の明確化**: レビュー対象のスキルディレクトリパスを明確に指定してください
2. **SKILL.md 必須**: SKILL.md が存在しない場合はエラーを報告します
3. **厳格かつ柔軟**: ベストプラクティスに厳格に準拠しますが、プロジェクト固有の事情も考慮します
4. **実行可能性**: 改善提案は具体的で実行可能なものを提供します
5. **比較例**: 修正前後の比較例を必ず含めます
